# Contributing to ICPC HUE

First off, thank you for considering contributing to ICPC HUE! 🚀

## 🤝 Code of Conduct

Help us keep the community specialized and welcoming. Please be respectful of others. We follow standard professional conduct.

## 🛠️ How Can I Contribute?

### 🐛 Reporting Bugs
- Use the [GitHub issue tracker](https://github.com/YUST777/icpchue_community/issues).
- Provide a clear, descriptive title.
- Include environment details (OS, Browser).
- List steps to reproduce and expected vs. actual behavior.

### ✨ Suggesting Enhancements
- Open a new issue with the `feature` label.
- Explain the "Why" and the "How".

### 👩‍💻 Pull Requests

1. **Branch Naming**:
   - `feat/feature-name` for new features.
   - `fix/bug-name` for bug fixes.
   - `docs/changes` for documentation.
2. **Commit Messages**: Use [Conventional Commits](https://www.conventionalcommits.org/) (e.g., `feat: add mirror mode support`).
3. **Environment**: Ensure your code works in the Docker environment.
4. **Linting**: Run `npm run lint` (or `bun lint` in `next-app`) before submitting.

## 🚀 Development Setup

1. **Clone**: `git clone https://github.com/YUST777/icpchue_community.git`
2. **Install**: `pnpm install` (Root and `next-app`).
3. **Docker**: `docker compose up -d` to launch the full stack.

For detailed architecture and internal docs, see [docs/rules/PROJECT_KNOWLEDGE_BASE.md](docs/rules/PROJECT_KNOWLEDGE_BASE.md).
